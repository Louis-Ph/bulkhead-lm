# Support

## Where to ask for help

- bug reports: open a GitHub issue with clear reproduction steps
- feature requests: open a GitHub issue describing the operator problem, not just the desired API shape
- security issues: follow [SECURITY.md](SECURITY.md) and do not report them publicly

## What to include

When asking for help, include:

- the commit or release you are using
- the relevant route or provider family
- sanitized config snippets
- sanitized logs or HTTP responses
- the exact command or request that reproduces the problem

## Support expectations

AegisLM is currently an early-stage project. Support is best-effort and review
bandwidth may vary.

Questions that are small, reproducible, and precise will be resolved faster than
broad architecture requests with no configuration context.
